### 1. Request Service (JobOffer Service)

**Публикация события:**
```python
# services/request_service/main.py
from event_bus.simple_event_bus import event_bus

@app.post("/api/requests")
def create_offer(request: CreateOfferRequest):
    offer_id = service.create_offer(command)
    event_bus.publish({"type": "offer_created", "offer_id": offer_id, "company": request.company})
    return {"request_id": offer_id}

**Bounded Context:** _Управление откликами на вакансии_

**Описание:**
Request Service отвечает за создание и управление откликами на вакансии, смену этапов отбора, добавление заметок и отклонение откликов. Сервис генерирует доменные события при изменении состояния отклика.

**API:**
- `POST /api/requests` — создание отклика
- `GET /api/requests/{id}` — получение отклика
- `POST /api/requests/{id}/assign-stage` — смена этапа
- `POST /api/requests/{id}/notes` — добавление заметки
- `POST /api/requests/{id}/reject` — отклонение отклика
- `GET /api/requests` — список активных откликов

**Публикуемые события:**
- `JobOfferCreated` — создан новый отклик
- `StageChanged` — изменён этап отбора
- `OfferRejected` — отклик отклонён
- `OfferReceived` — получен офер

### 2. Group Service (Project Service)

# services/group_service/main.py
@app.post("/api/groups")
def create_group(req: CreateGroupRequest):
    group_id = f"GRP-{uuid.uuid4().hex[:4].upper()}"
    groups[group_id] = Group(group_id, req.name, req.leader_id)
    event_bus.publish({"type": "group_created", "group_id": group_id})
    return {"group_id": group_id}


**Bounded Context:** _Управление проектами и группами_

**Описание:**
Group Service отвечает за создание и управление проектами/группами поиска работы. Позволяет объединять несколько откликов в один проект для удобного отслеживания.

**API:**
- `POST /api/groups` — создание группы
- `GET /api/groups/{id}` — получение группы
- `POST /api/groups/{id}/members` — добавление участника

**Публикуемые события:**
- `GroupCreated` — создана новая группа
- `GroupActivated` — группа активирована

### 3. Notification Service


# event_bus/simple_event_bus.py
class SimpleEventBus:
    def __init__(self):
        self._handlers = []
    def subscribe(self, handler):
        self._handlers.append(handler)
    def publish(self, event):
        for handler in self._handlers:
            handler(event)

event_bus = SimpleEventBus()


**Bounded Context:** _Уведомления пользователей_

**Описание:**
Notification Service подписывается на события от других сервисов и отправляет уведомления пользователям (email, push, консоль).

**Подписка на события:**
- `JobOfferCreated` — уведомление о создании отклика
- `StageChanged` — уведомление о смене этапа
- `OfferRejected` — уведомление об отклонении
- `GroupActivated` — уведомление об активации группы

### 4. Event Bus

# services/notification_service/main.py
from event_bus.simple_event_bus import event_bus

def handle_event(event):
    print(f"[Уведомление] {event.get('type')}: {event.get('offer_id')}")

event_bus.subscribe(handle_event)


**Тип:** In-Memory Event Bus (упрощённая версия без RabbitMQ)

**Описание:**
Event Bus используется для асинхронной коммуникации между сервисами. Сервисы публикуют события, а подписчики (Notification Service) обрабатывают их.

**События:**
| Событие | Публикует | Подписчик |
|---------|-----------|-----------|
| `offer_created` | Request Service | Notification Service |
| `stage_changed` | Request Service | Notification Service |
| `offer_rejected` | Request Service | Notification Service |
| `group_created` | Group Service | Notification Service |

# api_gateway/main.py
@app.api_route("/api/requests/{path:path}", methods=["GET", "POST"])
async def proxy_to_request(request: Request, path: str = ""):
    url = f"http://localhost:8001/api/requests/{path}"

@app.api_route("/api/groups/{path:path}", methods=["GET", "POST"])
async def proxy_to_group(request: Request, path: str = ""):
    url = f"http://localhost:8002/api/groups/{path}"
    return await proxy(request, url)


**Тип:** FastAPI Gateway

**Описание:**
API Gateway является единой точкой входа для всех клиентских запросов. Он маршрутизирует запросы к соответствующим сервисам.

**Маршрутизация:**
| Путь | Сервис | Порт |
|------|--------|------|
| `/api/requests/**` | Request Service | 8001 |
| `/api/groups/**` | Group Service | 8002 |

**Особенности:**
- Прозрачное проксирование запросов
- Обработка ошибок (Service Unavailable)
- Единый порт входа (8000)

## Вывод

В ходе лабораторной работы была реализована микросервисная архитектура для сервиса «Оффер где?».

Монолитная система была разделена на два независимых bounded context:
- **Request Service (JobOffer Service)** — управление откликами на вакансии
- **Group Service** — управление проектами и группами

Для асинхронного взаимодействия между сервисами был реализован **In-Memory Event Bus**, через который публикуются доменные события (`offer_created`, `stage_changed`, `group_created`). **Notification Service** выступает в роли подписчика и выводит уведомления в консоль.

Все сервисы объединены через **API Gateway**, который предоставляет единую точку входа на порту 8000 и маршрутизирует запросы к соответствующим сервисам.

Полученная архитектура демонстрирует преимущества микросервисного подхода: слабую связанность, независимое развитие сервисов и гибкость интеграции.
