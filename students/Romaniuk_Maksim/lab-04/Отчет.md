### 2. Command Handlers

---

#### **2.1 CreateJobOfferHandler**

**Описание:** Обработка создания нового отклика.

**Шаги:**
- генерация ID (OFFER-YYYY-NNNN)
- создание Value Object CompanyName
- создание агрегата JobOffer
- сохранение через репозиторий
- публикация доменных событий

**Файл:** `application/command/handlers/create_job_offer_handler.py`

```python
from datetime import datetime
import uuid
from domain.models.job_offer import JobOffer
from domain.value_objects.company_name import CompanyName

class CreateJobOfferHandler:
    def __init__(self, repository, publisher):
        self._repository = repository
        self._publisher = publisher

    def handle(self, command):
        year = datetime.now().year
        offer_id = f"OFFER-{year}-{uuid.uuid4().hex[:4].upper()}"
        company = CompanyName(command.company)

        job_offer = JobOffer(offer_id, company, command.position, command.user_id)

        self._repository.save(job_offer)
        self._publisher.publish(job_offer.events)
        job_offer.clear_events()

        return offer_id

#### **2.2 ChangeStageHandler**

**Описание:** Изменение этапа отбора.

**Шаги:**
- загрузка агрегата по ID
- проверка существования отклика
- проверка возможности смены этапа
- вызов метода change_stage() агрегата
- сохранение через репозиторий
- публикация доменных событий

**Файл:** `application/command/handlers/change_stage_handler.py`

```python
from domain.value_objects.stage_type import StageType

class ChangeStageHandler:
    def __init__(self, repository, publisher):
        self._repository = repository
        self._publisher = publisher

    def handle(self, command):
        job_offer = self._repository.find_by_id(command.offer_id)
        if not job_offer:
            raise ValueError(f"Job offer {command.offer_id} not found")

        if job_offer.is_rejected:
            raise ValueError(f"Cannot change stage of rejected offer")
        if job_offer.is_offer_received:
            raise ValueError(f"Cannot change stage of completed offer")

        new_stage = StageType(command.new_stage)
        old_stage = job_offer.current_stage.value

        if not job_offer.current_stage.can_transition_to(new_stage):
            raise ValueError(f"Cannot transition from {old_stage} to {new_stage.value}")

        job_offer.change_stage(new_stage, command.note)

        self._repository.save(job_offer)
        self._publisher.publish(job_offer.events)
        job_offer.clear_events()

        return {"offer_id": command.offer_id, "old_stage": old_stage, "new_stage": new_stage.value}

## Контрольные вопросы

1. **В чём разница между Command и Query?**  
   - Command изменяет состояние системы, Query только читает данные.

2. **Почему Command Handler возвращает только ID, а не весь объект?**  
   - Чтобы не нарушать CQRS и не смешивать чтение с записью; чтение выполняется отдельным Query.

3. **Где должна выполняться валидация: в команде, обработчике или доменной модели?**  
   - Примитивная валидация — в команде; бизнес-валидация — в доменной модели.

4. **Можно ли вызывать Query из Command Handler?**  
   - Нет, это нарушает принцип разделения CQRS.

5. **Зачем разделять Request DTO (от клиента) и Command (внутренний)?**  
   - Request DTO отражает внешний API, Command — внутреннюю модель приложения; они могут отличаться по структуре и правилам.
