## Контрольные вопросы

**В чём отличие Value Object от Entity?**  
Value Object не имеет ID и сравнивается по значениям. Entity имеет ID и изменяемое состояние.

**Почему Aggregate Root должен инкапсулировать доступ к внутренним сущностям?**  
Чтобы защитить инварианты и обеспечить консистентность данных.

**Какая роль Domain Events?**  
Фиксировать факты изменения состояния агрегата и уведомлять другие контексты.

**Как проверяете инварианты?**  
В конструкторах VO и в бизнес-методах агрегата.

**Почему Value Objects иммутабельны?**  
Чтобы гарантировать неизменность значений и упростить reasoning.
